Color Swatches

The colors should represent the actions they are used for. For example, a neutral color should be used for actions, where the state of the aepp doesn’t change (such as copying an address). The dramatic color should be used for more substantial changes (such as making a transaction).

  • .plain HEX#FFFFFF–RGB255/255/255–CMYK0/0/0/0–PANTONE 000C
  • .neutral HEX#F5F5F5–RGB245/245/a245–CMYK0/0/0/4–PANTONE 663C
  • .normal HEX#DCDCDC–RGB220/220/220–CMYK0/0/0/14–PANTONE CoolGray1C
  • .dramatic HEX#F7296E–RGB 247/41/110–CMYK0/83/55/3–PANTONE 213C
  • .exciting HEX#311B58–RGB49/27/88–CMYK44/69/0/65–PANTONE 2695C
  • .dark HEX#383838–RGB56/56/56–CMYK0/0/0/78–PANTONE BlackC
  • .darker HEX#1E1E1E–RGB30/30/30–CMYK0/0/0/88–PANTONE NeutralBlackC
  • .darkest HEX#000000–RGB0/0/0–CMYK0/0/0/100–PANTONE Black6C

Color Gradients

When gradients are used, we use the neutral gradients for backgrounds and saturated gradients for the active and inactive elements.

  • #FFFFFF .neutral-gradient #F1F4F7
  • #F22F70 .active-gradient #42255C
  • #6F53A1 .inactive-gradient #311B58

Color Ratios

Main interfaces should be relatively neutral and make the user "feel at home."

  • .plain
  • .neutral
  • .normal
  • .dramatic
  • .exciting

Color Ratios

Some interfaces can be darker, to differentiate from the main interfaces. Use this color ratio in menus, headers, footers, etc.

  • .exciting
  • .plain
  • .neutral
  • .dramatic
